## 实验室研究方向（其他方向待补充）
### 程序验证
程序验证旨在证明一个程序满足某些性质，比如正确性、内存安全性、（并发程序中的）活性等等。这对我们构建更安全、更可靠的软件系统至关重要。同时，我们的目标是产生能用机器检查的证明，而不仅仅是用纸笔推导完成证明。这意味着我们需要对待证明的语言、待证明的程序、待证明的性质、以及证明的过程进行形式化的建模，上述每一个环节都可以引出一些有意思的问题。

当前，验证领域已经出现一些颇具规模的结果，如经过验证的操作系统微内核 seL4，经过验证的 C 编译器 CompCert 等。不过我们希望最后能直接验证大家广泛使用的软件，如 Linux/gcc，这个过程显然会遇到极高的软件复杂性。如何处理大型系统中巨大的验证工作量，包括给每个函数写规约和证明，以及在遇到问题时检查并修改规约或代码，也是一个高难度的问题。近年 LLM 的兴起也为这方面的研究注入了新的活力。

胡老师组内的品牌项目 [CStar](https://gitee.com/cstarlang)，通过给 C 的源代码标注函数规约并插入证明块的方法完成 C 程序的证明。你可以选择参与 CStar 后端的建设，或者用 CStar 来证明一些常见的算法或数据结构，或者研究如何利用它证明基于 C 实现的真实系统。目前，CStar 对于并发程序验证的支持、CStar 证明过程中的 AI agent 辅助、CStar 工具链中的报错机制都有待完善，需要整个团队的同学们共同努力。

**由于 CStar 目前的文档不太成熟，所以我们需要在科研实习室中完善 “CStar 使用指南” section，以便后续使用科研实习室的同学们进行 troubleshooting。**

### 双向变换

双向变换的核心思想是 “Can Programming be Liberated from Unidirectional Style?”（能否从单向风格中解放编程？），研究如何使程序在” 正向” 和” 反向” 两个方向上都能转换，保持 roundtrip 属性。
